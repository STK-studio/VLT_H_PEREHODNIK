# H-Sport & VLT — link-hub

Статический сайт-«переходник» с официальными ссылками двух брендов (H-Sport и VLT).
Чистая статика: один `index.html` (весь CSS и JS внутри), без сборки и зависимостей.

## Файлы

| Файл | Назначение |
|------|------------|
| `index.html` | вся страница (HTML + CSS + JS) |
| `favicon.svg` | иконка вкладки |
| `og-image.png` | превью для соцсетей/мессенджеров (1200×630) |
| `.nojekyll` | отключает обработку Jekyll на GitHub Pages |

## Деплой на GitHub Pages

1. Создать **публичный** репозиторий, положить все файлы из этой папки в его корень.
2. Файл `.nojekyll` уже есть — он нужен, чтобы Pages раздавал сайт как есть.
3. Запушить в ветку `main`.
4. **Settings → Pages → Source:** «Deploy from a branch» → ветка `main`, папка `/ (root)` → **Save**.
5. Через ~1 минуту сайт будет на `https://<username>.github.io/<repo>/`.

Все пути к ассетам относительные — сайт корректно работает из под-пути `/<repo>/`.

## Локальный просмотр

Достаточно открыть `index.html` в браузере. Либо поднять локальный сервер:

```bash
python3 -m http.server 8000
# открыть http://localhost:8000
```

## Кастомный домен (опционально)

1. Добавить в корень файл `CNAME` с одной строкой — вашим доменом (например `links.example.ru`).
2. В DNS-провайдере прописать записи на GitHub Pages (A-записи на IP GitHub или CNAME на `<username>.github.io`).
3. Settings → Pages → Custom domain → указать домен, включить «Enforce HTTPS».

## Правка контента

Ссылки, слоганы и тексты — прямо в `index.html` (секции `.zone--h` и `.zone--vlt`).
Цвета брендов — в CSS-переменных: H-Sport `#EB6081`, VLT `#7E2436`, база `#0F0F12`.
